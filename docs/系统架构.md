# 系统架构

## 仓库目录

```text
interview/
|- web/                         Next.js 应用
|  |- src/app/                 路由、布局和路由处理器
|  |- src/components/          可复用界面组件（需要时创建）
|  |- src/features/            按产品功能组织的模块（需要时创建）
|  `- src/lib/                 小型框架适配器（需要时创建）
|- api/                         Python AI 与领域服务
|  |- src/interview_copilot/
|  |  |- api/                  HTTP 边界与鉴权路由
|  |  |- domain/               与厂商无关的领域模型
|  |  |- application/          账号、题库、RAG、面试与训练用例
|  |  |- infrastructure/       SQLAlchemy、pgvector 与审计适配
|  |  |- providers/            DeepSeek、豆包与百度适配器
|  |  |- sandbox/              本地受限代码执行适配器
|  |  |- speech/               语音票据、流式听写与厂商协议
|  |  `- agent_skills/         版本化训练 Skill 与评分标准
|  |- tests/                   测试
|  `- migrations/              Alembic 数据库迁移
|- infra/                       本地基础设施初始化
|- docs/                        工程规范与产品决策
|- docker-compose.yml
`- .env.example
```

标为“规划中”的目录只在出现第一个真实模块时创建，不使用空文件夹假装架构完整。

## 运行边界

```text
浏览器 -> Next.js Web/BFF -> FastAPI
                              |- 确定性应用服务
                              |- 有界训练状态图
                              |- 文档处理
                              |- 检索系统
                              |- 受限代码执行接口
                              |- 模型/语音适配器
                              `- PostgreSQL + pgvector / Redis（预留）
```

Next.js 不接触第三方厂商密钥。如果未来实时语音必须由浏览器直连，FastAPI 只签发权限受限、短时有效的凭证。

Coding Board 是模拟面试运行时的一部分。浏览器只提交版本化代码快照和运行请求，FastAPI 校验账号、会话、当前题目与幂等键后再调用执行器。当前本地执行器使用固定摘要的 Python 3.12 Docker 镜像，关闭网络并限制 CPU、内存、进程、时间、输出和文件系统权限。生产 Compose 默认关闭该执行器；生产环境必须使用独立执行服务，不能向 API 容器挂载 Docker Socket。

## 依赖方向

```text
API 边界 -> 应用层 -> 领域层
厂商/基础设施适配器 -> 应用层接口
```

领域层不能导入 FastAPI、SQLAlchemy、LangGraph、Redis 或任何厂商 SDK。应用层依赖协议接口，适配器负责实现接口。

## 数据归属

- PostgreSQL 是用户、会话、问答轮次、证据、报告、评分标准版本和语料元数据的唯一事实来源。
- pgvector 保存与来源记录关联的向量；向量属于可重新生成的派生数据。
- Redis 只用于缓存和协调，不能成为已完成会话的唯一存储。
- 当前不保存原始上传文件或录音，只保存规范化文本、来源元数据和用户确认后的回答文本。对象存储仅在出现原文件留存、异步 OCR 或合规归档需求时引入。
- 每条派生评价必须保存原始证据以及生成它的模型、Prompt 和评分标准版本。
- 事实核验还保存主张原话、核验状态、置信度、知识片段、来源链接和知识包版本；`uncertain` 不进入能力画像。

## 可观测性边界

- FastAPI 为每个 HTTP 请求接收或生成 `X-Request-ID`，响应头和错误响应都会返回同一标识。
- 访问日志只记录请求 ID、方法、路径、状态码和耗时，不记录查询参数、请求头、请求体、响应体、简历或回答正文。
- 请求校验错误不会回显 Pydantic 的原始 `input`；未知异常只记录异常类型，用户收到中文通用错误和请求 ID。
- `/health` 表示进程存活，`/ready` 同时检查 PostgreSQL 和 Redis；外部监控和告警接入仍由部署环境负责。

## AI 编排边界

- 模拟面试由确定性应用服务管理会话，模型只生成计划、追问决策和报告。
- 报告生成前可选运行受控 Verifier：只从已审核知识包检索，输出三态核验和 citation；检索或模型失败标记降级，不阻塞报告，也不自行判定事实错误。
- 结构化表达与业务 Sense 使用单个有界 LangGraph Agent；数据库仍是状态来源。
- Agent 只能调用注册并授权的只读工具。保存、删除、计费和状态迁移不能由模型决定。
- 三角 RAG 分别检索候选人、岗位和公共知识，生成阶段才组合，避免证据角色污染。
- 运行时 Skill 使用 metadata -> instructions/rubric 的渐进加载；MCP 等出现第二个真实消费者再引入。

## 后端语言决策

主后端采用 Python，因为产品核心是文档解析、LangGraph、检索和模型评测。并不禁止 Go；只有性能数据证明存在高并发媒体网关等独立瓶颈时，才能通过 ADR 引入 Go 服务，并明确运维负责人。
