# 待确认问题

以下决策会阻塞完整或可安全上线的实现。与项目负责人确认后，重要结论需要转为 ADR。

## 产品

- 登录使用账号和密码；当前阶段明确不做邮箱验证和找回密码。已实现 Redis 账号标识限流：标识先规范化并做 SHA-256 摘要，默认 15 分钟内 8 次失败后限制，成功登录清除计数；Redis 故障时记录结构化安全降级日志并放行。生产上线前仍需根据真实反向代理链确定可信来源地址解析、账号与来源地址的组合策略，以及安全降级告警渠道，避免伪造地址、全站误限流或反向锁死目标账号。
- 训练草稿默认保存 7 天，用户可导出数据并通过当前密码确认永久注销 PostgreSQL 账号数据；仍需确定过期任务执行频率和备份清除方式。
- 项目负责人希望默认允许匿名数据用于改进。该设计上线前必须完成隐私/合规审查，界面需要明确说明用途、范围、撤回方式，禁止把原始简历、完整录音、凭据或可识别数据直接作为训练语料。

## 语音

- 已确认在线语音合成 `wss://tts-api.xfyun.cn/v2/tts` 与语音听写 `wss://iat-api.xfyun.cn/v2/iat`。TTS `xiaoyan` 真实返回 MP3；IAT 真实完成鉴权、PCM 分帧和最终帧响应。仍需确认免费额度、到期时间和生产并发上限。
- 短时 HMAC 语音票据已使用 Redis `SET NX EX` 原子消费 nonce，实现严格的一次性使用；Redis 不可用时拒绝语音连接，不静默绕过重放保护。
- TTS 当前生成完整 MP3 后播放；项目负责人已决定不继续深挖讯飞体验，后续优先评测并替换为豆包语音 Provider。
- `edge-tts` 只作为开发试听工具，不作为生产 TTS；当前正式实现使用科大讯飞在线语音合成。
- 豆包实时语音需要取得当前官方接口、音色和额度后，用同一中文面试样本评测首包延迟、自然度、打断、稳定性和单场成本，再替换讯飞 Provider。

## 已核验厂商契约

- DeepSeek 官方文档于 2026-07-16 核验：OpenAI base URL `https://api.deepseek.com` 支持 `tools` Function Calling；账号模型列表实际返回 `deepseek-v4-flash` 和 `deepseek-v4-pro`。`deepseek-v4-flash` 已真实返回 `finish_reason=tool_calls` 以及结构正确的函数名和 JSON 参数。JSON Output 也已通过真实请求核验。实测同时发送 `tools` 与 `response_format=json_object` 可能抑制工具调用，因此训练 Agent 使用“工具阶段关闭 JSON Mode，最终格式化阶段关闭工具”的两阶段协议。当前模型默认开启 thinking；结构化题库与训练规划请求按官方 `thinking.type=disabled` 关闭思考模式，避免输出预算被推理内容耗尽。
- 豆包于 2026-07-12 通过真实请求核验：`POST https://ark.cn-beijing.volces.com/api/v3/embeddings/multimodal`，请求 `model=ep-20260611172909-x4pkc`、文本 input、`dimensions=1024`，成功返回 1024 维向量；响应报告底层模型为 `doubao-embedding-vision-250615`。
