# 安全基线

## 高风险改动

以下内容投入生产前必须由项目负责人明确审查：

- 登录、鉴权、会话 Cookie 和账号恢复；
- 处理简历、录音、逐字稿等个人数据；
- 外部文件解析、OCR、压缩包解压或 URL 抓取；
- 代码执行和面试 Coding 沙箱；
- 厂商密钥、签名 URL、Webhook 校验和计费；
- 破坏性数据库迁移以及数据保留/删除任务；
- 模型工具调用、自动执行外部操作和语料入库。

## 强制控制

- 默认拒绝访问，服务端必须验证数据归属。
- 代理层和应用层都要限制请求体及文件大小。
- 生产环境根据文件内容验证 MIME，不能只信扩展名。
- 解析器和代码执行必须隔离并限制资源，绝不执行上传文件中的宏或嵌入内容。
- 根据输出位置正确转义，并配置严格的内容安全策略 CSP。
- 浏览器状态变更接口要防护 CSRF，并校验允许的 Origin。
- 登录、上传、面试轮次、TTS 和高成本模型接口都要限流并设置额度。
- 未来 JD 链接导入必须防 SSRF：限制协议、阻止私有/链路本地地址、限制跳转和响应大小。
- 私有数据传输和存储都要加密；收集生产录音前必须确定保存及删除规则。
- 审计访问和删除事件，但日志不能包含敏感正文。

## 当前限制

- PDF 和 DOCX 已检查文件签名/容器结构，上传限制 20MB；扫描 PDF 的百度 OCR 额外限制为 10MB、20 页。解析器尚未放入隔离 Worker，也未接入恶意文件检测，因此上传接口仍不满足公网生产要求。
- 账号、HttpOnly 会话、资源归属、CSRF Origin 校验、Argon2 密码哈希、登录失败限流、当前密码二次确认注销和账号数据导出已实现；当前阶段明确不做邮箱验证和找回密码，生产上线前仍需根据部署规模评估异常登录告警。
- HTTP 访问日志使用请求 ID 关联，仅记录方法、路径、状态和耗时；查询参数、请求/响应正文和认证头不得进入日志。校验错误不得回显原始输入。
- 数据导出不包含密码哈希、会话令牌和厂商密钥；账号注销会删除 PostgreSQL 中的账号关联数据。对象存储原始文件与备份副本的清理仍待保留策略确定后接入。
- 面试中的未提交回答仅进入当前标签页的 `sessionStorage` 以支持断网和刷新恢复，不写入长期 `localStorage`；问题变化、提交成功或结束面试时清除。提交结果未知时必须沿用原幂等 ID，禁止生成新 ID 重复写入。
- 科大讯飞 TTS/IAT、百度 OCR 已完成真实契约请求；阿里云百炼 Qwen 和 Embedding 的生产额度、并发、熔断和全局成本上限仍需部署前核验。
- 实时打断使用 Redis 按用户、会话和问题限频；其他高成本模型接口仍需要统一的用户级额度策略。
- 语音 WebSocket 票据绑定用户与训练会话，短时有效，并在会话归属校验后通过 Redis 原子消费 nonce；票据重放或 Redis 不可用时拒绝连接。该边界不依赖具体 TTS/IAT 厂商。
- Coding Board 只允许当前账号拥有且处于算法阶段的面试会话访问。代码快照和运行请求使用客户端幂等 ID，并限制每题最多 100 个版本、每版本最多 30 次运行。
- 本地代码执行固定为 Python 3.12 镜像摘要，使用非 root 用户、只读根文件系统、无网络、无新增权限、独立 IPC，并限制 CPU、内存、进程、文件描述符、临时目录、执行时间和输出大小；超时、输出超限或请求取消都会终止容器。
- 生产 Compose 默认关闭本地 Docker 执行器。上线前必须部署独立执行服务并完成隔离审查，禁止把 Docker Socket 挂载到 API 容器。
