# ADR-0002：资料来源链与用户确认式求职记忆

## 状态

已接受，2026-07-16。

## 背景

资料导入如果只保存生成后的题目，无法判断题库是否覆盖原文，也无法解释题目依据。长期求职过程需要保存目标和计划，但把对话或模型推断直接当作“记忆”会产生隐私、错误传播和不可控写入风险。

## 决定

- 题库以 `question_documents` 保存规范化文本、内容哈希、文件名版本、生成状态、覆盖率、模型和 Prompt 版本，不保存原始二进制文件。
- 资料按语义片段分批生成题目；`question_evidence` 保存题目对应的片段编号、标题路径和逐字原文。没有可核验证据的模型输出不入库。
- 相同用户的相同内容按哈希去重；同名但内容变化产生新版本。删除资料级联删除题目与证据，并清理对应检索索引。
- 结构化训练只能读取用户明确选择且经应用层校验归属的题目来源；Agent 工具不能扩展本轮资料 UUID 集合。
- 长期记忆只包括用户明确提交的求职画像和周计划。能力画像可以提供确定性建议，但不自动写入；Agent 没有保存、更新或删除工具。
- 账号导出包含规范化资料、证据、求职画像和周计划；不包含原文件、录音或不受限对话记忆。

## 备选方案

- 只保存生成题目：实现简单，但不能做覆盖率、证据追溯和可靠重生成，因此拒绝。
- 保存完整对话作为通用记忆：上下文丰富，但隐私范围、事实有效期和模型误写难以控制，因此拒绝。
- 由 Agent 自动维护画像：交互成本低，但写入边界不可审计，也会把推断误当用户意图，因此拒绝。

## 影响与回退条件

导入资料会增加数据库文本、证据和多次模型调用成本；大文件需要分批生成。只有在经过隐私审查、提供逐项查看/撤回能力并有明确事实有效期后，才重新评估自动记忆。只有出现原文件留存、异步 OCR 或合规归档需求时，才引入对象存储。
