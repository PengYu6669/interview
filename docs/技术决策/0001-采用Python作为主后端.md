# ADR 0001：采用 Python 作为主后端

## 状态

已接受

## 背景

后端工作主要是文档解析、检索、LangGraph 编排、模型评测和第三方服务接入。Python 和 Go 都能提供可靠的网络服务，但对这些工作负载的生态支持不同。

## 决定

使用 Python 与 FastAPI 作为主后端，并保持领域协议与具体厂商无关。只有性能分析确认某类工作负载需要独立服务时，才考虑引入 Go。

## 影响

- 可以直接使用成熟的文档和 AI 工具链。
- CPU 密集的 OCR 和解析工作不能阻塞异步请求线程，必须放入 Worker。
- 运行期类型安全依赖 mypy、Pydantic、测试和清晰边界共同保障。
- 未来引入 Go 时，需要单独定义服务协议、ADR、可观测性和运维责任。
