# ADR-0003：公共题库与个性化面试出题策略

## 状态

已实施，2026-07-20。

当前实现包括 `user/admin` 角色、受控授权命令、管理员候选题列表与结构化编辑、发布前字段和证据校验、发布/下线、公共题库默认入口、个人资料二级入口，以及模拟面试的简历/JD/题库混合输入。管理员导入复用现有异步题库任务，任务归属当前管理员；所有管理员可以继续审核其他管理员创建的候选题，普通用户私有题不进入管理列表。

## 背景

学习中心当前允许普通用户上传 PDF、Word、Markdown 或文本资料，系统依次执行文档解析、知识点映射、向量索引、分批出题和原文证据修复。默认生成 30 题预计需要 600 秒，题量更大时预计 900 秒。虽然任务在后台运行，但把这条高成本、长等待链路放在学习中心主入口，会让首次使用者在看到学习价值前先承担资料选择和等待成本。

模拟面试已经读取简历、JD、目标岗位、职级、轮次、训练重点、用户选题和三角检索证据，并把生成后的面试计划保存为会话事实。现有能力适合承担个性化出题，但不能完全替代经过审核的公共技术题、编程题和评分依据。

项目已有 `questions.published`、`questions.owner_user_id`、资料来源链、题目证据和个人学习进度，但没有管理员角色、服务端管理权限、候选题审核和发布入口。

## 决定

### 产品边界

- 学习中心以可立即学习的公共题库为默认入口，突出搜索、筛选、题目集和复习进度。
- 普通用户资料导入继续保留，但只放在“我的题目”下作为二级能力，不再作为学习中心主行动。
- 管理员使用独立后台上传资料。生成结果先作为私有候选题保存，审核后逐题发布到公共题库。
- 第一版只有 `user` 和 `admin` 两种数据库角色，不实现编辑、审核员等多级角色。
- 管理页面隐藏不是授权措施；所有管理 API 必须在 FastAPI 边界验证 `admin` 角色。

### 公共题库生命周期

```text
管理员上传资料
-> 后台解析、知识点映射、索引和生成候选题
-> 候选题保持 published=false
-> 管理员查看来源、编辑题目
-> 发布前校验字段和来源证据
-> published=true，进入公共题库
-> 必要时下线，保留题目及用户学习进度
```

- AI 生成完成不等于内容审核通过，不允许自动发布。
- 发布动作不复制题目，直接改变同一题目的公开状态，避免进度和引用关系分叉。
- 来源资料生成的题目必须有可核验原文证据才能发布。
- 下线只改变公开状态，不物理删除用户学习记录或已经引用该题目的历史面试会话。
- 管理员上传仍使用现有异步任务和失败恢复机制，页面可以离开后再返回。

### 模拟面试出题

- 项目深挖和行为题根据简历事实动态生成。
- 岗位场景题根据 JD、职级和轮次动态生成。
- 技术基础和系统设计题以公共题库的考察意图、答案结构和常见错误作为锚点，再结合候选人材料改写。
- 用户显式选择题目时优先覆盖其考察意图，但不逐字复制参考答案。
- 编程题优先使用经过验证的公共题和测试用例；AI 临时生成的编程题必须通过结构、资源限制和测试用例校验后才能进入面试。
- 面试计划只生成一次并持久化；运行时只针对用户回答生成必要追问，不在每轮重新生成整套题。
- 模型不得把简历、JD、题库或检索内容中的文本当成系统指令执行。

## 数据与接口

- `users.role` 保存 `user` 或 `admin`，默认 `user`，由受控命令 `python -m interview_copilot.admin_cli grant <username>` 显式授予管理员角色；`revoke` 撤回权限。
- `UserProfile` 和 Web 登录态返回角色，用于导航展示；最终权限仍以后端记录为准。
- 管理 API 提供候选题列表、详情、编辑、发布和下线操作；管理界面通过现有 `/v1/questions/import` 创建归属当前管理员的异步导入任务。
- 普通题库 API 继续只返回 `published=true` 的题目；个人题库 API 继续只返回当前用户拥有的题目。
- 账号导出不得因为管理员功能而包含其他用户资料。

## 权限与安全

- 普通用户访问管理 API 返回 403，未登录访问返回 401。
- 管理员不能通过请求参数指定其他用户身份；操作者来自会话认证结果。
- 管理员上传资料仍受文件大小、格式、OCR、文本长度和 Prompt 注入防护约束。
- 不记录原始文件、凭据或完整请求正文；继续只保存规范化文本和证据。
- 公共题目的来源必须经过版权与公开使用授权确认。技术实现保留来源字段，但授权确认属于上线前运营流程。

## 实施阶段

1. 增加用户角色迁移、管理员依赖和管理员授权命令。
2. 增加候选题列表、管理员导入、发布和下线 API 及 BFF。
3. 增加管理页面，展示异步任务状态、候选题来源和发布状态。
4. 调整学习中心信息架构，将个人导入移动到“我的题目”。
5. 保持并测试模拟面试的混合出题输入；后续单独评测动态题与公共锚点题的比例。

## 测试要求

- 认证：注册用户默认为 `user`；登录和 `/me` 返回角色；无效角色不能进入领域模型。
- 授权：未登录访问管理 API 返回 401；普通用户返回 403；管理员可以访问。
- 数据隔离：普通用户不能看到其他用户候选题或资料；公共题库只返回已发布题。
- 发布：有来源资料但无证据的题目发布失败；合格候选题可以发布和下线；重复操作幂等。
- 导入：管理员上传继续覆盖大小、格式、生成失败、任务恢复和超时路径。
- 前端：非管理员无管理入口；管理页面对任务处理中、失败、无候选题和发布状态都有明确反馈。
- 面试：无选题时仍依据简历和 JD 生成；有选题时传入题库上下文；越权题目被草稿服务拒绝。
- 回归：API pytest、Ruff、mypy、Web lint/build 和 Compose 配置全部通过。

## 交付标准

- 普通用户进入学习中心后无需等待生成即可浏览公共题库。
- 个人资料导入不出现在学习中心首屏主行动中，只在“我的题目”提供。
- 管理员能够上传资料、离开页面、返回查看生成结果，并逐题发布或下线。
- 普通用户无法通过直接访问 URL 或构造请求调用管理能力。
- AI 生成题不会未经审核进入公共题库。
- 模拟面试继续保存模型、Prompt 版本和实际生成计划，失败时不创建假成功会话。
- 数据库迁移具有明确降级路径：删除角色约束和列前，所有账号恢复为普通用户语义。

## 影响与回退

该决定增加管理员权限和内容运营成本，但把高延迟生成从普通用户主路径转移到后台，并保留少数高级用户的私有资料能力。若后台运营无法持续提供足够公共题目，可以重新提升个人导入入口，但不能取消私有数据隔离和人工发布门槛。若动态面试题质量不稳定，可以提高公共锚点题占比，而无需回退学习中心和权限模型。
