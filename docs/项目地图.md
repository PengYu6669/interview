# InterviewCopilot 项目地图

> 只记录当前真实实现。核心链路、事实来源或外部服务变化时更新，不作为开发日志。

## 一句话说明

面向技术岗位候选人的学习与训练 Web：从个人材料、岗位要求和公共知识中检索证据，支持题库学习、真实模拟面试、结构化表达、业务 Sense 训练，并把有原句证据的结果沉淀到成长档案。

## 运行边界

```text
Browser -> Next.js Web/BFF -> FastAPI
                            |- deterministic services
                            |- bounded training Agent / LangGraph
                            |- RAG indexing and retrieval
                            |- DeepSeek / Doubao Embedding / Baidu OCR / speech provider
                            `- PostgreSQL + pgvector
```

- Next.js 负责页面、浏览器状态、登录展示和 BFF，不保存厂商密钥。
- FastAPI 负责文档、检索、训练编排、评价、语音票据和厂商适配。
- PostgreSQL 是账号、会话、回答、报告、评价和审计的事实来源；Redis 只承担登录失败限流等可丢失的协调状态，不是核心事实来源。
- 不保存原始录音；语音识别文本由用户确认提交后才作为回答保存。

## 当前核心链路

### 学习中心

- 入口：`/questions`、`/questions/[slug]`。
- 调用链：Next BFF -> `api/questions.py` -> `QuestionWorkflowService` -> PostgreSQL；带引用问答通过混合 RAG 检索所选题目，再调用 DeepSeek。
- 状态与事实来源：题目、Markdown、来源、学习状态和聊天记录均在 PostgreSQL。
- 失败表现：解析、检索或模型失败返回中文错误，不伪造答案或引用。

### 结构化表达与业务 Sense

- 入口：`/training/new?mode=...` -> `/training/[sessionId]`。
- 调用链：BFF -> `api/coaching.py` -> `CoachingService` -> `TrainingCoachAgent` -> versioned Skill -> DeepSeek Tool Calling -> candidate/job/knowledge retrieval tools。
- 状态与事实来源：`coaching_sessions`、`coaching_turns`；最多 5 轮，回答使用客户端幂等键。
- 不变量：评价只使用用户原句证据；证据不足不能给分；模型只读工具，保存与状态更新由应用服务执行。
- 失败表现：会话归属、状态、模型结构和工具调用均校验；工具调用另存脱敏审计。

### 模拟面试

- 入口：`/setup` -> `/interview?session=...` -> `/report?session=...`。
- 调用链：材料解析/结构化提取 -> 三角 RAG -> DeepSeek 生成计划 -> PostgreSQL 会话 -> 多轮回答与面试官决策 -> 证据化报告。
- 状态与事实来源：面试会话、轮次和报告在 PostgreSQL；浏览器缓存只用于未提交步骤恢复。
- 语音：短时 HMAC 票据连接 FastAPI WebSocket；当前讯飞 IAT/TTS 在 Provider 后，后续可替换豆包。
- 失败表现：登录、材料归属、会话状态、票据来源和回答幂等均有明确失败路径。

### 成长档案

- 入口：`/history`、`/profile`。
- 调用链：BFF -> 历史/画像 API -> 面试报告与专项训练回答证据聚合。
- 边界：模拟面试报告形成 K 线和技术能力矩阵；专项训练以 1-5 级证据独立聚合，不能混成同一量尺。
- 报告异议：用户可请求 DeepSeek 独立复核单项评分，或明确排除出长期技能矩阵；原报告不改写，复核结论、理由、模型和 Prompt 版本单独留痕。`uncertain` 视为证据不足，不进入技能画像。
- 复训：面试弱项回到 `/setup`；表达与业务弱项回到对应 `/training/new`。

## RAG 与 Agent

- 三角 RAG：`candidate` 表示候选人事实，`job` 表示岗位上下文，`knowledge` 表示题库和已审核公共知识；事实核验只允许检索 `curated_knowledge_pack`。
- 索引：规范化 -> 质量告警 -> 标题/段落/代码块感知切块 -> 豆包 1024 维向量 -> pgvector。
- 检索：向量、PostgreSQL 全文检索、trigram 和精确子串并行候选 -> 独立阈值 -> RRF 融合 -> 带来源证据。
- Verifier：报告阶段提取最多 3 条可验证主张 -> 检索已审核知识包 -> DeepSeek 批量输出 `supported / contradicted / uncertain` -> 校验引用编号和置信度 -> 保存版本化 citation。无证据或低于 0.8 的冲突自动为 `uncertain`。
- Agent：只用于结构化表达与业务 Sense 的受控训练；LangGraph 负责有限分支，不替代数据库状态。
- Tools：三类只读检索工具；服务端注入用户身份，包含白名单、超时、预算、结果上限和脱敏审计。
- Skills：运行时只先加载 metadata，需要训练时再加载 `SKILL.md` 与 rubric；Skill、Prompt 和模型版本随会话保存。
- MCP：当前没有第二个真实消费者，暂不封装，避免增加无收益的协议层。

## 实现状态

- 已从代码确认：账号密码登录、Redis 登录失败限流、题库编辑/导入/检索问答、三角 RAG、模拟面试状态与报告、报告异议/独立复核/画像排除、两类专项训练、语音转写基线、成长档案、能力画像、工具审计、公共知识包和检索评测。
- 登录限流按规范化账号标识的 SHA-256 摘要计数，默认 15 分钟内 8 次失败后限制；成功登录清除计数。Redis 故障时记录结构化安全降级日志并放行，避免认证服务整体不可用。
- 已真实核验：DeepSeek Tool Calling/JSON、报告单项复核、报告 Verifier（QA 场次真实得到 1 条 `supported` 与 1 条 citation）、豆包 Embedding、PostgreSQL/pgvector、讯飞 IAT/TTS；检索评测集 v2 包含正例、脏查询、易混淆和无答案共 16 例，结果为 `Recall@5=1.0`、`MRR=0.896`、Top-1 `0.833`、无答案准确率 `1.0`。
- 已实现：System Design Board 的结构化节点、连线、拖动、删除、自动保存、乐观版本控制与冲突处理；白板快照可按账号和面试会话持久化。
- 已实现：报告生成读取面试会话最新白板快照，保存报告级结构化副本，并在报告页提供只读回放；白板不会因后续编辑而改变历史报告。
- 已实现：账号数据导出包含报告关联的白板快照；账号删除沿面试会话级联清理白板数据。
- 已实现：题库学习状态包含间隔复习调度（复习间隔、连续掌握次数、上次复习和到期时间），仍以 `user_question_progress` 为唯一事实来源。
- 已实现：题库提供账号级“待复习”入口，按到期时间返回公共题目或本人题目，并复用原有题目详情和面试选择流程。
- 已实现：API/Web 生产镜像、Next standalone 构建、非 root 运行、生产 Compose 内部网络、启动迁移、存活/就绪检查和环境变量边界；镜像已真实构建，生产依赖安全审计为 0 漏洞。
- 已实现：FastAPI 请求 ID、结构化访问日志和统一错误关联；日志不记录查询参数或正文，校验错误不回显不可信原始输入。
- 已实现：PostgreSQL custom-format 备份与显式确认恢复脚本；真实备份已通过 `pg_restore --list` 验证，外部备份目标和保留周期仍待服务器环境确定。
- 已实现：模拟面试和专项训练语音票据一次性消费；Redis 原子阻止重放，Redis 故障时安全拒绝连接，逻辑与讯飞/豆包 Provider 解耦。
- 尚未实现：Coding Board 及隔离执行沙箱、生产反向代理/外部备份同步/日志告警、豆包实时语音替换。
- 当前明确不做：数字人、邮箱验证、密码找回、原文件对象存储、通用 MCP；出现真实需求或第二个消费者时重新评估。

## 下一步只做什么

下一项：先确定 Coding Board 的隔离执行边界，包括支持语言、CPU、内存、执行时间、文件系统和网络权限，再实现受限代码运行。功能与本地验收完成前不继续生产部署；豆包实时语音替换需等当前官方接口、音色和额度核验后推进。
